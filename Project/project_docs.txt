1 Introduction
1.1 Purpose
This Software Requirements Specification (SRS) document describes the functional and
non-functional requirements of theCareer Development and Placement Manage-
ment System (CDPMS). The document is intended to be used by members of the
project team involved in the design, development, testing, and verification of the system.
Unless otherwise stated, all requirements specified in this document are considered
high priority and are mandatory for the successful implementation of the system.
1.2 Project Scope and Product Features
1.2.1 Project Overview
Universities conduct placement activities involving students, recruiters, and placement
officers. At present, these activities are largely managed using emails and spreadsheets,
which makes the process time-consuming, error-prone, and difficult to monitor.
The Career Development and Placement Management System (CDPMS) aims to pro-
vide a centralized, web-based solution to automate student registrations, company job
postings, application processing, eligibility verification, interview scheduling, and place-
ment status tracking.
The system is designed as a web application to ensure platform independence and
easy accessibility without requiring any local software installation. The system supports
multiple levels of access to accommodate students, companies, and placement officers.
1.2.2 Project Scope
The CDPMS is divided into three major functional components: Student Management,
Recruitment Management, and Placement Reporting.
Student ManagementThe system allows students to register by providing personal
details, academic records, skills, and resumes. Each student is assigned a unique place-
ment ID. Students can apply for job openings, view the status of their applications, and
download offer letters once made available.
3
--- Page 5 ---
Recruitment ManagementCompanies can register and post job openings with de-
tails such as eligibility criteria, salary package, application deadlines, and recruitment
stages. The system automatically verifies student eligibility prior to job application.
Placement officers schedule recruitment activities including written tests, group discus-
sions, and interviews.
Placement ReportingThe system provides reports such as company-wise placement
statistics, student-wise placement details, and department-wise placement percentages.
Feedback mechanisms for both students and companies are included to improve the place-
ment process.

2 Overall Description
2.1 Product Perspective
The Career Development and Placement Management System (CDPMS) is a standalone,
web-based application designed to manage the complete placement lifecycle within a uni-
versity. The system consolidates placement-related data into a single platform, improving
efficiency, transparency, and accuracy.
The system is modular in nature, allowing future enhancements such as advanced an-
alytics, automated notifications, and integration with existing university systems. Role-
based access control ensures that users interact with the system according to their re-
sponsibilities.
2.2 User Classes and Characteristics

letters.
Placement Officer Placement officers manage the overall placement
process, verify eligibility, schedule recruitment
stages, monitor candidate progress, and generate
reports.
